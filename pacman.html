PAC-MAN GODMODE

  • Project-type:Group Project for the course CS7632-Game AI
  • Where:Georgia Tech
  • My Role: Design,Development,Presentation and Video

PACMAN

Pac-man is a maze chase video game in which the objective is to have a yellow circular character Pac-man navigate through a maze consuming food on it's way while avoiding ghosts each of which pursue it. When all the food is consumed the game ends and the player wins. When Pac-man comes in contact with a ghost he loses a life. When all lives are lost the game ends and the player loses.

GOAL

To create a Reinforcement Learning agent to replace the player's default Keyboard agent transiently, and take optimal actions and maximize rewards. Secondly we aimed to trigger this mode through some specific user action within a set of constraints. Our initial idea was to have the player(Pac-Man) consume a specially implemented super power pellet to trigger the mode. The player would benefit by offloading the decision making on the bot when the game becomes too difficult or tricky to play. We saw this will give a two fold advantage:

  • It would give players incentive to look for the super power pellet and hence will add an interesting factor in the game.
  • Secondly, this can be helpful when the game is getting too tricky or complex and would allow the player to offload the stress for a little while, giving the feeling of safe fallback.

Algorithm and Implementation

Reinforcement Learning

  • The computer or the agent employs trial and error to come up with a solution to the problem.
  • To get the machine to do what the programmer wants, the artificial intelligence gets either rewards or penalties for the actions it performs.
  • Its goal is to maximize the total reward.

Q-learning

  • For Q-learning in the training phase, the agent tries an action for each state. It then records the rewards its getting. The Q value is calculated from the formed state action table.
  • In the testing phase, the agent would apply the action that had maximum Q value for the given state in the state action table.

Approximate Q-learning

  • The grid used Pacman can take 107 positions, similarly the two ghosts each can take 107 positions ultimately creating a huge state space.Therefore we used an Approximate Q-learning approach instead of standard Q-learning.
  • The idea is to learn a reward function as a linear combination of features.
  • Features: 'eats-food', 'bias', 'closest-food', '#-of-ghosts-1-step-away'
  • We trained the agent using 50 training episodes. Using the final trained weights we tested our game using 10 test games.
